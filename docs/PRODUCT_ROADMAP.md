# Π立方企业服务中心 - 产品规划文档

## 一、业务概述

Π立方企业服务中心是一家综合性企业服务公司，核心业务为代理记账，同时提供工商注册/变更、知识产权、法律服务、设计服务、资质代办等多元化服务。

### 服务分类

| 服务类型 | 具体业务 | 服务周期 | 收费模式 |
|----------|----------|----------|----------|
| **财税服务** | 代理记账 | 持续性（月/季/年） | 周期性收费 |
| **工商服务** | 公司注册、公司变更、公司注销、年报公示 | 一次性 | 按项目收费 |
| **知识产权** | 商标注册、专利申请、软著登记、版权登记 | 一次性 | 按项目收费 |
| **法律服务** | 法律咨询、合同起草/审核、律师函 | 按次/按小时 | 按次/计时 |
| **设计服务** | Logo设计、VI设计、画册/名片设计 | 一次性 | 按项目收费 |
| **资质代办** | 经营许可证、资质认证、行业许可 | 一次性 | 按项目收费 |

---

## 二、系统定位

管理系统是给**员工使用**的，核心目的是：
- 员工用系统来**接单干活**
- 公司根据员工的完成情况与员工**算账（分润）**

### 角色分工

| 角色 | 核心诉求 | 常用功能 |
|------|----------|----------|
| **销售** | 拉单、看业绩、拿提成 | 客户管理、新建工单、发起工单、我的业绩 |
| **会计/专员** | 接单、干活、拿提成 | 接单大厅、我的工单、记账工作台、我的收入 |
| **管理员** | 统筹、派单、算账 | 全部工单、人员管理、结算管理、数据报表 |

### 4.1 工单完整生命周期

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           工单生命周期                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   客户下单（客户端）                                                         │
│        │                                                                    │
│        ▼                                                                    │
│   【待对接】── 销售接收通知，联系客户                                         │
│        │                                                                    │
│        ▼                                                                    │
│   【待确认价格】── 销售修改价格，客户确认                                     │
│        │                                                                    │
│        ▼                                                                    │
│   【待发单】── 销售设置分润，发布工单                                         │
│        │                                                                    │
│        ├─────────────────────────────────────┐                             │
│        ▼                                     ▼                             │
│   【发单到大厅】                       【指派给指定人】                        │
│   （所有人可见抢单）                   （直接分配给某人）                      │
│        │                                     │                             │
│        └──────────────────┬──────────────────┘                             │
│                           ▼                                                │
│   【待接单】── 专员确认接单                                                  │
│        │                                                                    │
│        ▼                                                                    │
│   【进行中】── 专员执行工单，系统监控进度                                     │
│        │                                                                    │
│        ▼                                                                    │
│   【待确认】── 专员提交完成，销售确认                                         │
│        │                                                                    │
│        ▼                                                                    │
│   【已完成】── 进入结算流程                                                   │
│        │                                                                    │
│        ▼                                                                    │
│   【已结算】── 分润到账                                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 状态说明

| 状态 | 触发条件 | 可见范围 | 可执行操作 |
|------|----------|----------|------------|
| 待对接 | 客户提交订单 | 销售本人 | 销售对接客户 |
| 待确认价格 | 销售对接完成 | 销售、客户 | 销售改价、客户确认 |
| 待发单 | 客户确认价格 | 销售本人 | 销售设置分润、发单 |
| 待接单 | 销售发单 | 大厅所有人或指定人 | 抢单、接受指派 |
| 进行中 | 接单成功 | 执行人、销售、管理员 | 更新进度、上传附件 |
| 待确认 | 执行人提交完成 | 销售、客户 | 销售确认、客户确认 |
| 已完成 | 确认完成 | 所有人 | 查看详情、评价 |
| 已结算 | 财务结算 | 销售本人、执行人 | 查看收入 |

### 4.3 分润模型

```
工单金额: ¥2000
├── 销售分润: 30% → ¥600（拉单的人）
├── 执行人分润: 50% → ¥1000（干活的人）
└── 平台分润: 20% → ¥400（公司）
```

**分润规则说明**：
- 销售在发单时可调整分润比例
- 默认比例可在系统管理中配置
- 指派工单时，执行人分润比例可协商

## 五、系统监控机制

### 5.1 进度监控

工单执行过程中，系统自动监控：

| 监控项 | 规则 | 处理方式 |
|--------|------|----------|
| 截止日期 | 提前1天/3天提醒 | 消息通知执行人 |
| 超期预警 | 超过截止日期 | 标红提醒、通知销售 |
| 进度停滞 | 超过N天未更新 | 通知执行人、销售 |

### 5.2 自动处理

| 场景 | 规则 | 处理方式 |
|------|------|----------|
| 超时无人接单 | 发单后24小时无接单 | 通知销售重新分配 |
| 完成超时确认 | 提交后48小时未确认 | 自动确认完成 |
| 客户无响应 | 价格确认超时 | 通知销售跟进 |

## 六、数据模型设计

### 6.1 核心表结构

```sql
-- 工单表
CREATE TABLE work_orders (
  id VARCHAR(32) PRIMARY KEY,
  type VARCHAR(32) NOT NULL,           -- 工单类型
  title VARCHAR(255) NOT NULL,         -- 工单标题
  customer_id VARCHAR(32) NOT NULL,    -- 客户ID
  description TEXT,                    -- 详细描述
  
  status VARCHAR(32) NOT NULL,         -- 状态
  progress INT DEFAULT 0,              -- 进度百分比
  current_step VARCHAR(255),           -- 当前步骤
  
  amount INT NOT NULL,                 -- 工单金额（分）
  cost INT DEFAULT 0,                  -- 成本（分）
  
  sales_id VARCHAR(32) NOT NULL,       -- 销售ID
  handler_id VARCHAR(32),              -- 执行人ID
  
  deadline TIMESTAMP,                  -- 截止日期
  completed_at TIMESTAMP,              -- 完成时间
  
  sales_share_rate INT DEFAULT 30,     -- 销售分润比例
  handler_share_rate INT DEFAULT 50,   -- 执行人分润比例
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 工单进度记录
CREATE TABLE work_order_progress (
  id VARCHAR(32) PRIMARY KEY,
  work_order_id VARCHAR(32) NOT NULL,
  step_name VARCHAR(255) NOT NULL,
  status VARCHAR(32) NOT NULL,         -- completed/in_progress/pending
  note TEXT,
  completed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 沟通记录
CREATE TABLE work_order_communications (
  id VARCHAR(32) PRIMARY KEY,
  work_order_id VARCHAR(32) NOT NULL,
  user_id VARCHAR(32) NOT NULL,
  content TEXT NOT NULL,
  attachments JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 9.3 线索管理

| 功能 | 说明 |
