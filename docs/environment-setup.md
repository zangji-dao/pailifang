# 环境配置说明

## 概述

本项目采用**代码统一、环境变量驱动**的架构设计，确保沙箱环境和生产环境使用完全相同的代码。

## 两种运行模式

### 模式一：安全隔离模式（默认）

```
┌─────────────────────┐     ┌─────────────────────┐
│      沙箱环境       │     │      生产环境       │
├─────────────────────┤     ├─────────────────────┤
│ 数据库: pi_cube_dev │     │ 数据库: pi_cube     │
│ 数据: 测试数据      │     │ 数据: 生产数据      │
└─────────────────────┘     └─────────────────────┘

✅ 安全：沙箱操作不影响生产
✅ 隔离：独立的测试数据库
```

**启动方式**：
```bash
./deploy.sh sandbox
# 或
./scripts/dev.sh
```

---

### 模式二：完全一致模式

```
┌─────────────────────┐     ┌─────────────────────┐
│      沙箱环境       │     │      生产环境       │
├─────────────────────┤     ├─────────────────────┤
│ 数据库: pi_cube     │ ←───│ 数据库: pi_cube     │
│ 数据: 生产数据      │     │ 数据: 生产数据      │
└─────────────────────┘     └─────────────────────┘

✅ 完全一致：与生产环境相同
⚠️  风险：测试操作会影响生产数据！
```

**启动方式**：
```bash
USE_PROD_DB=true ./deploy.sh sandbox
# 或
USE_PROD_DB=true ./scripts/dev.sh
```

---

## 数据同步

### 从生产同步数据到测试数据库

```bash
# 完整同步（结构 + 数据）
./scripts/sync-db.sh

# 仅同步结构
# 选择选项 2

# 仅同步数据
# 选择选项 3
```

### 定时自动同步（推荐）

在生产服务器上配置 cron：

```bash
# 每天凌晨 3 点同步
0 3 * * * /var/www/pi-cube/scripts/sync-db.sh <<< "1" >> /var/log/pi-cube/sync.log 2>&1
```

## 使用场景

| 场景 | 推荐模式 | 说明 |
|------|---------|------|
| 日常开发 | 安全隔离 | 测试新功能，不影响生产 |
| Bug 复现 | 完全一致 | 用生产数据定位问题 |
| 上线前验证 | 数据同步 | 先同步生产数据，再在隔离模式测试 |
| 紧急修复 | 完全一致 | 快速定位并修复线上问题 |

---

## 安全建议

1. **默认使用隔离模式**：避免误操作影响生产
2. **完全一致模式需确认**：启动时会提示确认
3. **敏感操作前备份**：使用 `pg_dump` 备份生产数据
4. **权限分离**：测试数据库和生产数据库使用不同权限

---

## 快速参考

```bash
# 安全隔离模式（默认）
./deploy.sh sandbox

# 完全一致模式
USE_PROD_DB=true ./deploy.sh sandbox

# 同步生产数据到测试库
./scripts/sync-db.sh

# 初始化测试数据库
./scripts/init-dev-db.sh

# 部署到生产
./deploy.sh production

# 同步代码到生产服务器
./deploy.sh sync
```
