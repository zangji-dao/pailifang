# 环境配置说明

## 概述

本项目采用**沙箱与生产完全一致**的架构，代码自动适配环境，无需手动修改配置。

## 端口配置（自动适配）

```
┌─────────────────────────────────────────────────────────────────┐
│                     端口自动适配策略                            │
├─────────────────────────────────────────────────────────────────┤
│  环境    │  前端端口        │  后端端口    │  说明              │
├──────────┼─────────────────┼─────────────┼────────────────────┤
│  沙箱    │  5000 (平台强制) │  4001       │  DEPLOY_RUN_PORT   │
│  生产    │  4000 (可配置)   │  4001       │  PORT 环境变量     │
└─────────────────────────────────────────────────────────────────┘

代码自动判断：
- 检测到 DEPLOY_RUN_PORT → 沙箱环境，使用 5000
- 检测到 PORT 或默认 → 生产环境，使用 4000
```

## 架构图

```
┌─────────────────────┐     ┌─────────────────────┐

├─────────────────────┤     ├─────────────────────┤

```

## 环境判断

代码自动通过以下方式判断环境：

| 判断条件 | 环境 |
|---------|------|
| `COZE_PROJECT_ENV=DEV` | 沙箱 |
| `COZE_PROJECT_DOMAIN_DEFAULT` 含 `dev.coze.site` | 沙箱 |
| 其他 | 生产 |

## 快速开始

### 沙箱环境

```bash
./deploy.sh sandbox
```

### 生产环境

```bash
# 默认使用 4000 端口
./deploy.sh production

# 或指定端口
PORT=8080 ./deploy.sh production
```

### 同步代码到生产

```bash
./deploy.sh sync
```

## 敏感信息

通过环境变量注入，不在代码中硬编码：

| 环境变量 | 说明 |
|---------|------|
| `PG_PASSWORD` | 数据库密码 |
| `ALIPAY_PRIVATE_KEY` | 支付宝私钥 |
| `ALIPAY_PUBLIC_KEY` | 支付宝公钥 |

## 目录结构

```
.
├── config/
│   └── env.ts              # 前端环境配置
├── backend/
│   └── src/
│       ├── config/
│       │   └── env.ts      # 后端环境配置
│       └── index.ts        # 后端入口
├── src/
│   └── lib/
│       └── apiClient.ts    # 前端 API 客户端
├── scripts/
│   ├── dev.sh              # 沙箱启动脚本
│   └── start.sh            # 生产启动脚本（自动适配端口）
└── deploy.sh               # 统一部署脚本
```

## 注意事项

1. **端口自动适配**：代码自动判断环境，无需手动修改端口配置
2. **沙箱端口固定**：平台强制使用 5000，无法更改
3. **生产端口可配置**：通过 `PORT` 环境变量配置，默认 4000
