# 环境配置说明

## 概述

本项目采用**沙箱与生产完全一致**的架构：

- 同一份代码
- 同一个数据库
- 同一套配置

```
┌─────────────────────────────────────────────────────────────────┐
│                        Git 仓库                                 │
│              同一份代码，沙箱和生产自动适配                      │
└─────────────────────────────────────────────────────────────────┘
                    │                           │
                    ▼                           ▼
        ┌─────────────────────┐     ┌─────────────────────┐
        │      沙箱环境       │     │      生产环境       │
        ├─────────────────────┤     ├─────────────────────┤
        │ COZE_PROJECT_ENV=DEV│     │ COZE_PROJECT_ENV=PROD│
        │ 前端: 5000          │     │ 前端: 4000          │
        │ 后端: 4001          │     │ 后端: 4001          │
        │ 数据库: pi_cube ◀───┼─────┼─▶ 数据库: pi_cube   │
        └─────────────────────┘     └─────────────────────┘
                    │                           │
                    └─────────── 完全一致 ───────┘
```

## 环境判断

代码自动通过 `COZE_PROJECT_ENV` 判断环境：

| 环境变量 | 环境 | 说明 |
|---------|------|------|
| `DEV` | 沙箱 | 开发调试 |
| `PROD` | 生产 | 线上服务 |

## 快速开始

### 沙箱环境

```bash
# 启动服务（自动启动前后端）
./deploy.sh sandbox
```

### 生产环境

```bash
# 在生产服务器上执行
./deploy.sh production
```

### 同步代码到生产

```bash
# 从沙箱同步代码到生产服务器
./deploy.sh sync
```

## 敏感信息

通过环境变量注入，不在代码中硬编码：

| 环境变量 | 说明 |
|---------|------|
| `PG_PASSWORD` | 数据库密码 |
| `ALIPAY_PRIVATE_KEY` | 支付宝私钥 |
| `ALIPAY_PUBLIC_KEY` | 支付宝公钥 |

```

│       └── index.ts        # 后端入口

│   └── lib/
│       └── apiClient.ts    # 前端 API 客户端

│   ├── dev.sh              # 沙箱启动脚本
│   └── start.sh            # 生产启动脚本
└── deploy.sh               # 统一部署脚本
```

## 注意事项

1. **沙箱操作会影响生产数据**：因为共用数据库，沙箱的测试操作会直接修改生产数据
2. **谨慎删除操作**：测试时避免删除数据，或先备份
3. **上线前备份**：重要操作前使用 `pg_dump` 备份数据库
