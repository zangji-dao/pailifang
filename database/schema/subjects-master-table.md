# 会计科目汇总主表

> 设计思路：一张主表囊括所有准则的科目，用布尔字段标记各准则适用性

---

## 一、数据库表结构设计

```sql
-- 会计科目主表
CREATE TABLE accounting_subjects_master (
    id SERIAL PRIMARY KEY,
    code VARCHAR(10) NOT NULL COMMENT '科目编码',
    name VARCHAR(50) NOT NULL COMMENT '科目名称',
    category VARCHAR(10) NOT NULL COMMENT '科目类别：资产/负债/权益/成本/损益',
    level INT NOT NULL DEFAULT 1 COMMENT '科目级次：1-一级，2-二级，3-三级',
    parent_code VARCHAR(10) COMMENT '上级科目编码',
    balance_direction VARCHAR(10) NOT NULL COMMENT '余额方向：借/贷',
    
    -- 准则适用标记
    for_small_enterprise BOOLEAN DEFAULT FALSE COMMENT '小企业会计准则',
    for_enterprise BOOLEAN DEFAULT FALSE COMMENT '企业会计准则',
    for_nonprofit BOOLEAN DEFAULT FALSE COMMENT '民间非营利组织',
    for_cooperative BOOLEAN DEFAULT FALSE COMMENT '农民专业合作社',
    for_union BOOLEAN DEFAULT FALSE COMMENT '工会会计',
    for_rural_collective BOOLEAN DEFAULT FALSE COMMENT '农村集体经济组织',
    
    -- 纳税人类型标记（仅增值税相关科目）
    for_general_taxpayer BOOLEAN DEFAULT TRUE COMMENT '一般纳税人可用',
    for_small_taxpayer BOOLEAN DEFAULT TRUE COMMENT '小规模纳税人可用',
    
    -- 其他属性
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    description TEXT COMMENT '科目说明',
    sort_order INT COMMENT '排序号',
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(code)
);

-- 创建索引
CREATE INDEX idx_subjects_category ON accounting_subjects_master(category);
CREATE INDEX idx_subjects_level ON accounting_subjects_master(level);
CREATE INDEX idx_subjects_small_ent ON accounting_subjects_master(for_small_enterprise);
CREATE INDEX idx_subjects_enterprise ON accounting_subjects_master(for_enterprise);
```

### 2. 同编码不同名称科目处理

部分科目在不同准则下编码相同但名称不同，处理方式：

```sql
-- 方案一：名称字段存储多个名称，用JSON
{
  "names": {
    "small_enterprise": "应付利润",
    "enterprise": "应付股利",
    "rural_collective": "应付利润"
  }
}

-- 方案二：按准则优先级取名称
-- 优先级：企业准则 > 小企业准则 > 其他

-- 方案三：在账套初始化时确定具体名称
-- 根据账套选择的准则，从名称映射表取对应名称
```
