# 财务会计科目体系数据库设计

## 一、数据库表结构概览

```
┌─────────────────────────┐
│  accounting_standards   │  会计准则表
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│  accounting_subjects    │  会计科目表（核心）
└───────────┬─────────────┘
            │
    ┌───────┴───────┐
    ▼               ▼
┌─────────┐   ┌──────────────┐
│报表模板表│   │增值税科目配置│
└────┬────┘   └──────────────┘
     │
     ▼
┌──────────────┐
│ report_items │  报表项目表
└──────────────┘
```

## 二、核心表说明

### 1. 会计准则表 (accounting_standards)

| 字段 | 说明 |
|-----|------|
| id | 准则标识 |
| name | 准则名称 |
| code | 准则编码 |
| description | 准则说明 |

**预设数据**：
- 小企业会计准则
- 企业会计准则
- 民间非营利组织会计制度
- 农民专业合作社财务制度
- 工会会计制度
- 农村集体经济组织核算制度

### 2. 会计科目表 (accounting_subjects)

| 字段 | 说明 | 示例 |
|-----|------|------|
| standard_id | 所属准则 | small_enterprise |
| category_id | 科目类别 | assets/liabilities/equity/cost/profit_loss |
| code | 科目编码 | 1001 |
| name | 科目名称 | 库存现金 |
| parent_code | 上级科目 | NULL（一级） |
| level | 科目级别 | 1-5级 |
| is_leaf | 是否末级 | true/false |
| balance_direction | 余额方向 | debit（借方）/ credit（贷方） |
| has_auxiliary | 是否辅助核算 | true/false |
| auxiliary_types | 辅助核算类型 | ["customer","project"] |

### 3. 报表模板表 (report_templates)

| 字段 | 说明 |
|-----|------|
| standard_id | 所属准则 |
| report_type | 报表类型 |
| name | 报表名称 |

### 4. 报表项目表 (report_items)

| 字段 | 说明 |
|-----|------|
| template_id | 所属报表模板 |
| item_name | 项目名称 |
| formula | 计算公式 |
| subject_codes | 对应科目编码 |

### 5. 增值税科目配置表 (vat_subject_config)

针对一般纳税人/小规模纳税人的增值税科目差异配置。

## 三、科目编码规则

```
资产类：1xxx
负债类：2xxx
所有者权益类：3xxx
成本类：4xxx
损益类：5xxx（收入）、6xxx（费用）

示例：
1001    库存现金         （一级）
1002    银行存款         （一级）
100201  银行存款-工商银行 （二级）
100202  银行存款-建设银行 （二级）
```

## 五、企业会计准则 vs 小企业会计准则 差异

### 科目数量对比
| 准则 | 一级科目数 | 说明 |
|-----|-----------|------|
| 小企业会计准则 | ~60个 | 简化版 |
| 企业会计准则 | ~150个 | 完整版 |

### 主要差异科目

| 业务 | 企业会计准则 | 小企业会计准则 |
|-----|-------------|---------------|
| 短期投资 | 交易性金融资产 | 短期投资 |
| 长期投资 | 持有至到期投资、可供出售金融资产 | 长期债券投资 |
| 减值准备 | 坏账准备、存货跌价准备等 | 可不计提 |
| 所得税 | 递延所得税资产/负债 | 无 |
| 公允价值 | 公允价值变动损益 | 无 |

## 六、创建账套时的科目初始化流程

```
1. 用户选择会计准则 + 纳税人类型
           ↓
2. 从模板表加载对应准则的科目体系
           ↓
3. 根据纳税人类型调整增值税科目
   - 一般纳税人：加载多栏账科目（进项/销项/转出等）
   - 小规模纳税人：加载简单科目
           ↓
4. 将科目数据复制到用户账套科目表
           ↓
5. 用户可自定义添加下级明细科目
```
